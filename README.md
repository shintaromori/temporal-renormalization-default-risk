# Temporal Coarse-Graining of Latent Default-Probability Paths

This repository contains the analysis code for the paper

**Temporal Coarse-Graining of Latent Default-Probability Paths Generates Effective Default Correlation**

by Shintaro Mori.

The purpose of this repository is to provide a minimal executable workflow for the main analyses in the paper. The empirical default data used in the paper are not included because they are derived from proprietary historical default datasets and cannot be redistributed.

Instead, this repository provides a low-fidelity synthetic sample dataset, `data/SAMPLE.csv`, with the same basic column structure and monthly frequency. The sample data are provided only for code execution and workflow demonstration.

Results obtained from `SAMPLE.csv` are not expected to reproduce the numerical values reported in the paper.

## Notebooks

The notebooks should be run in the following order.

### `02.ipynb`

This notebook reads `data/SAMPLE.csv` and constructs non-overlapping temporally aggregated default-count data.

Main output:

```text
pdata/SP_monthly_ALL_kmonth.csv
```

This file is used by both `03.ipynb` and `04.ipynb`.

### `03.ipynb`

This notebook reproduces the Section II workflow of the paper:

* empirical variance scaling of the monthly-equivalent default rate;
* Bayesian estimation of the monthly OU--Binomial state-space model;
* temporal coarse-graining of posterior default-probability paths;
* posterior predictive variance decomposition;
* autocorrelation diagnostics;
* effective mixing distributions induced by temporal coarse-graining.

Main output includes:

```text
pdata/idata_ou_binom_k1.nc
```

This posterior file is reused by `04.ipynb`.

### `04.ipynb`

This notebook reproduces the Section III workflow of the paper:

* direct fitting of OU--Binomial, OU--Davis--Lo, and OU--Vasicek models at each aggregation scale;
* covariance-share diagnostics for residual same-period dependence;
* WAIC and per-block elpd comparisons;
* renormalized fitting conditional on coarse-grained posterior default-probability paths;
* residual-parameter diagnostics for the OU--Davis--Lo and OU--Vasicek extensions;
* generation of the main figures and summary tables for Section III.

The notebook reuses

```text
pdata/idata_ou_binom_k1.nc
```

generated by `03.ipynb`.

## Data availability

The empirical default data analyzed in the paper are not included in this repository. They are derived from proprietary historical default datasets and therefore cannot be publicly shared.

The file `data/SAMPLE.csv` is independently generated low-fidelity synthetic data. It is not derived from, fitted to, or calibrated to the proprietary empirical default data used in the paper.

The sample data are provided only to make the notebooks executable and to demonstrate the workflow. Therefore, the numerical values, figures, and tables obtained from `SAMPLE.csv` will not match the empirical results reported in the paper.

Researchers with access to comparable default-count data can reproduce the empirical workflow by replacing `data/SAMPLE.csv` with their own monthly default-count data having the same column structure.

## Input data format

The notebooks expect `data/SAMPLE.csv` to have the following columns:

```text
date,bonds,defaulted
```

where:

* `date` is the monthly date;
* `bonds` is the number of obligors;
* `defaulted` is the number of defaults in that month.

## Installation

The code was developed in Python. A typical environment can be prepared with:

```bash
pip install -r requirements.txt
```

The main packages used are:

* `numpy`
* `pandas`
* `scipy`
* `matplotlib`
* `pymc`
* `arviz`
* `pytensor`
* `jupyter`

Depending on the local environment, installation of `pymc` may require additional backend dependencies.

## Running the analysis

After installing the required packages, start Jupyter:

```bash
jupyter notebook
```

Then run the notebooks in order:

```text
02.ipynb
03.ipynb
04.ipynb
```

The notebooks write intermediate files to

```text
pdata/
```

and figures to

```text
figures/
```

For a clean run, remove existing files in `pdata/` and `figures/` before executing the notebooks again.

## Reproducibility note

The Bayesian analyses use Markov chain Monte Carlo sampling. Small numerical differences may occur across platforms, package versions, and random seeds.

The sample data are intended for workflow testing only. The numerical values obtained from `SAMPLE.csv` should not be compared with the empirical results in the paper.
